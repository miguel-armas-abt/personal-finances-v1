participant frontend
participant personal-finances-v1
participant mongo-db

frontend->personal-finances-v1: POST /auth/refresh (refreshToken)

personal-finances-v1->personal-finances-v1: hash presented refresh token

personal-finances-v1->mongo-db: [db.refresh_tokens] find token by hash
mongo-db-->personal-finances-v1: **return** refresh token record

alt token is missing, expired or revoked
    personal-finances-v1->mongo-db: [db.refresh_tokens] revoke token family if reuse is suspected
    mongo-db-->personal-finances-v1: **return** revoked tokens

    personal-finances-v1-->frontend: 401 Unauthorized
else token is valid
    personal-finances-v1->mongo-db: [db.users] find enabled user by ID
    mongo-db-->personal-finances-v1: **return** user

    alt user is disabled
        personal-finances-v1-->frontend: 401 Unauthorized
    else user is enabled
        personal-finances-v1->mongo-db: [db.refresh_tokens] revoke current refresh token
        mongo-db-->personal-finances-v1: **return** revoked refresh token

        personal-finances-v1->mongo-db: [db.refresh_tokens] persist rotated refresh token hash
        mongo-db-->personal-finances-v1: **return** new refresh token record

        personal-finances-v1->personal-finances-v1: generate internal access token

        personal-finances-v1-->frontend: 200 OK (accessToken, expiresIn, refreshToken, user)
    end
end
